A border paints a 1- or 2-pixel-thick ring around the entire panel perimeter. Four styles are available: an animated **rainbow chase** that sweeps a hue gradient per pixel around the edge, a **color_cycle** that pulses the entire border as one animated hue, a **constant color** that frames the content in a steady solid color, and **lightbulbs** that paint discrete bulb sprites for a classic marquee aesthetic ‚Äî lit bulbs can optionally use a static rainbow coloring. Both `rainbow` and `color_cycle` accept optional `from` / `to` RGB colors to confine the animation to a specific hue arc instead of sweeping the full spectrum. All styles paint at physical panel resolution ‚Äî they bypass the `ScaledCanvas` block-expansion path and call `SetPixel` on the real underlying canvas, so a 1-pixel border is always exactly one LED wide regardless of scale.

## Border styles

| Style         | Shorthand                | Per-pixel color?            | Animates? |
| ------------- | ------------------------ | --------------------------- | --------- |
| `rainbow`     | `border = "rainbow"`     | yes ‚Äî hue chase             | yes       |
| `color_cycle` | `border = "color_cycle"` | no ‚Äî whole border one hue   | yes       |
| `constant`    | `border = [r,g,b]`       | no ‚Äî uniform                | no        |
| `lightbulbs`  | `border = "lightbulbs"`  | optional ‚Äî rainbow or solid | yes       |

### Rainbow chase

The rainbow chase indexes each perimeter pixel by its clockwise position around the edge, then advances the entire pattern by `speed` degrees per engine tick (50 ms). The result is a smooth hue gradient that rotates continuously. `char_offset` controls how tightly the hue cycles around the perimeter: a higher value tiles more full rainbows along the edge.

The simplest way to enable it is the string shorthand:

<TomlExample
  title="String shorthand"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = "rainbow"`}
/>

To tune the three knobs, switch to an inline table:

<TomlExample
  title="With tuning"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = {style = "rainbow", thickness = 2, speed = 4, char_offset = 6}`}
/>

**`speed`** (default `4`): degrees of hue rotation per engine tick. At 50 ms per tick, the default produces about one full revolution every 12 seconds on the bigsign's 640-pixel perimeter. Double it to `8` for a noticeably faster sweep.

**`char_offset`** (default `6`): hue spacing between adjacent perimeter pixels. At `6` and 640 pixels, roughly 10 full rainbows tile around the bigsign edge. Reduce it toward `1` for a slow gradient wash; raise it above `10` for a tighter, flickery pattern.

**`thickness`** (default `1`): `1` is a single-pixel ring; `2` is two concentric rings. Thickness 2 reads better at viewing distance. The two rings share the same hue index sequence, so the inner ring continues where the outer ring ends rather than repeating.

#### Restricting the hue arc

Add `from` and `to` RGB colors to confine the chase to a specific hue arc ‚Äî the chase will tile only within those two hues rather than sweeping the full 360¬∞ spectrum. The engine always takes the shorter arc between the two hues.

<DemoGif
  src="/demos-pinned/border-rainbow-range.gif"
  caption="red ‚Üí amber arc at `speed = 3`, `thickness = 2` ‚Äî chase tiles only within the warm fire band, never entering green or blue"
/>

<TomlExample
  code={`# Warm fire arc ‚Äî red to amber, 2-ring border
border = {style = "rainbow", from = [220, 30, 0], to = [255, 180, 0], speed = 3, thickness = 2}`}
/>

`char_offset` controls how many full arc-widths tile around the perimeter ‚Äî the same density semantics as the full-wheel chase, just measured in arc-degrees instead of 360¬∞. A narrow arc (60¬∞) with `char_offset = 6` produces roughly the same number of tiles as the full-wheel chase, because the hue spacing is relative to the arc width. Achromatic `from`/`to` colors (gray, white, black) are rejected since their hue is undefined. Same-hue `from`/`to` is also rejected ‚Äî use `border = [r, g, b]` for a static color.

#### Common patterns

**Fire / warm tones** ‚Äî keep the chase in the red-to-amber band:

<TomlExample
  code={`border = {style = "rainbow", from = [220, 30, 0], to = [255, 180, 0], speed = 3, thickness = 2}
# Red ‚Üí amber chase, 2-ring border`}
/>

**Cool ocean sweep** ‚Äî blue-to-cyan arc for a cold-light feel:

<TomlExample
  code={`border = {style = "rainbow", from = [0, 80, 255], to = [0, 220, 255], speed = 4, thickness = 1}
# Deep blue ‚Üí cyan chase`}
/>

**Rainbow chase + arc-restricted font_color** ‚Äî pair a full rainbow border with a brand-palette text color:

<TomlExample
  code={`[[playlist.section.widget]]
type = "message"
text = "Open"
font_color = {style = "color_cycle", from = [255, 176, 240], to = [189, 169, 234], speed = 3}
border = {style = "rainbow", from = [255, 176, 240], to = [189, 169, 234], speed = 3, thickness = 2}`}
/>

### Color cycle

A color_cycle border paints every perimeter pixel the same color per frame, and that color cycles through the hue wheel over time. Unlike the rainbow chase ‚Äî which gives each pixel its own hue offset ‚Äî color_cycle pulses the border as a single unified color. The effect is a slow wash of light that shifts hue without any chasing motion.

<DemoGif
  src="/demos-pinned/border-color_cycle.gif"
  caption="`color_cycle` border at `speed = 5`, `thickness = 2` ‚Äî entire perimeter pulses as one hue with no per-pixel chase"
/>

String shorthand uses the full 360¬∞ wheel at the default speed:

<TomlExample
  title="String shorthand"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = "color_cycle"`}
/>

Tune `speed` and `thickness` via an inline table:

<TomlExample
  title="With tuning"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = {style = "color_cycle", speed = 5, thickness = 2}`}
/>

**`speed`** (default `5`): degrees of hue rotation per engine tick. Higher = faster pulse.

**`thickness`** (default `1`): `1` is a single-pixel ring; `2` is two concentric rings.

#### Restricting the hue arc

Add `from` and `to` RGB colors to confine the cycle to a specific hue arc ‚Äî the border will pulse between those two colors rather than sweeping the full spectrum. The engine always takes the shorter arc between the two hues.

<DemoGif
  src="/demos-pinned/border-color_cycle-range.gif"
  caption="soft pink ‚Üí lavender arc at `speed = 2` ‚Äî border pulses only within the pink/purple band, never drifting into green or yellow"
/>

<TomlExample
  code={`# Soft pink ‚Üí lavender arc ‚Äî stays within the brand palette
border = {style = "color_cycle", from = [255, 176, 240], to = [189, 169, 234], speed = 2, thickness = 2}`}
/>

`speed = 0` is rejected ‚Äî use `border = [r, g, b]` for a static color. Achromatic `from`/`to` colors (gray, white, black) are also rejected since their hue is undefined.

#### Common patterns

**Brand-palette pulse** ‚Äî lock the border to your two brand hues so it never wanders off-palette:

<TomlExample
  code={`border = {style = "color_cycle", from = [189, 169, 234], to = [255, 176, 240], speed = 2, thickness = 2}
# Lavender ‚Üí soft pink pulse, 2-ring border`}
/>

**Contrasting border from text** ‚Äî pair a color_cycle border with a complementary arc on `font_color`:

<TomlExample
  code={`[[playlist.section.widget]]
type = "message"
text = "Open"
font_color = {style = "color_cycle", from = [255, 200, 50], to = [255, 100, 0], speed = 4}   # amber arc
border   = {style = "color_cycle", from = [120, 230, 255], to = [50, 100, 255], speed = 4}   # blue arc`}
/>

### Constant

A constant border is a solid-color ring with no animation. It is `frame_invariant`, so the engine's static-text fast path applies: the border is painted once and the widget sleeps for the hold duration rather than redrawing every tick.

<DemoGif
  src="/demos-pinned/border-constant.gif"
  caption="constant magenta ring around the panel perimeter, text inside"
/>

Use an RGB list as a shorthand:

<TomlExample
  title="RGB shorthand"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = [255, 100, 200]   # constant magenta`}
/>

To add thickness, switch to an inline table:

<TomlExample
  title="With thickness"
  code={`[[playlist.section.widget]]
type = "message"
text = "Hello"
border = {style = "constant", color = [255, 100, 200], thickness = 2}`}
/>

## Where it works

| Widget type                                                                         | `border`?                  |
| ----------------------------------------------------------------------------------- | -------------------------- |
| `message`                                                                           | yes                        |
| `countdown`                                                                         | yes                        |
| `two_row`                                                                           | yes                        |
| `gif`                                                                               | yes                        |
| `image`                                                                             | yes                        |
| `weather`, `rss_feed`, `mlb`, `mlb_standings`, `coinbase`, `coingecko`, `etherscan` | no ‚Äî raises at config load |

Data widgets render custom layouts and a perimeter border is not a meaningful concept for them. The config validator catches any misplaced `border =` on these types at startup so the error surfaces immediately rather than silently doing nothing.

## Continuous-phase behavior

Both `rainbow` and `color_cycle` borders have `restart_on_visit = false` ‚Äî their phase counters advance continuously and never reset between `loop_count` repetitions. If a section repeats with `loop_count = 2`, the border picks up exactly where it left off at the end of the first loop rather than snapping back to the start. The border also composes cleanly with other effects: a `TickerMessage` can carry `animation = "typewriter"` and `border = "rainbow"` simultaneously, and each effect tracks its own independent counter. See [frame counters](/reference/frame-counters/) for the full model of restart-on-visit versus continuous-phase behavior.

## Lightbulbs

The `lightbulbs` style paints discrete NxN bulb sprites around the perimeter ‚Äî the classic Vegas-marquee aesthetic. Each bulb is a fixed-size square sprite at one of ~100 (bigsign-default) clockwise-ordered positions; the animation modes flip which bulbs are "lit" vs "unlit" per frame. Both colors paint every frame, so "off" bulbs glow dimly like physical incandescent bulbs without power ‚Äî the default is a warm-white lit color over a dim warm-orange unlit color.

The simplest form is the string shorthand:

<TomlExample
  code={`[[section]]
[[section.widget]]
type = "message"
text = "HELLO"
border = "lightbulbs"`}
/>

Defaults: `mode = "chase"`, `bulb_size = 3` on big panels (auto-falls back to `1` on panels shorter than 32 physical pixels), `gap = 3`, warm-white lit / dim-warm-orange unlit colors.

### Modes

- **`chase`** ‚Äî every Nth bulb (default 3rd) is lit; the lit set walks clockwise (or `direction = "ccw"`) around the perimeter, advancing one bulb position every `speed_frames` engine ticks. Classic traveling-marquee look.

<DemoGif
  src="/demos-pinned/border-lightbulbs-chase.gif"
  caption="`chase` ‚Äî warm-white marquee, the lit window travels clockwise"
/>

- **`alternate`** ‚Äî even/odd bulbs flip on each phase. Half the bulbs lit at any time, switching every `speed_frames` ticks. Looks like a shimmer or twinkle without directional motion.

<DemoGif
  src="/demos-pinned/border-lightbulbs-alternate.gif"
  caption="`alternate` ‚Äî even/odd bulbs flip in place, a directionless twinkle"
/>

- **`unison`** ‚Äî all bulbs share state. All lit, then all unlit, alternating every `speed_frames` ticks. Loud attention-grabber.

<DemoGif
  src="/demos-pinned/border-lightbulbs-unison.gif"
  caption="`unison` ‚Äî every bulb blinks on and off together"
/>

### Full table form

<TomlExample
  code={`border = { style = "lightbulbs",
           mode = "chase",                # "chase" | "alternate" | "unison"
           bulb_size = 3,                 # optional; auto from panel height
           gap = 3,                       # pixels between bulb edges
           lit_color = [255, 220, 140],   # warm white default; or "rainbow"
           unlit_color = [40, 20, 0],     # dim warm-orange default
           speed_frames = 2,              # frames per state transition
           chase_density = 3,             # only used by mode="chase"
           direction = "cw",              # "cw" | "ccw", only used by mode="chase"
           hue_wraps = 1.0 }              # only used when lit_color = "rainbow"`}
/>

`speed_frames` defaults differ by mode: `2` for chase (~100 ms per bulb step), `5` for alternate (~250 ms per toggle), `8` for unison (~400 ms per blink). Higher values slow the animation; the engine ticks at 50 ms.

### Rainbow bulbs

Set `lit_color = "rainbow"` to color each lit bulb by its position around the perimeter ‚Äî a full hue spectrum wrapped around the ring. The hues are **static in space**: they do not rotate over time, so with `mode = "chase"` the lit window travels across a fixed string of colored bulbs, exactly like a strand of party or Christmas lights. Unlit bulbs keep their `unlit_color` glow.

<DemoGif
  src="/demos-pinned/border-lightbulbs-rainbow.gif"
  caption={
    '`lit_color = "rainbow"` with `mode = "chase"` ‚Äî a traveling lit window over a fixed rainbow string'
  }
/>

<TomlExample
  code={`[[playlist.section.widget]]
type = "message"
text = "PARTY"
border = { style = "lightbulbs", mode = "chase", lit_color = "rainbow" }`}
/>

**`hue_wraps`** (default `1.0`): how many full spectra tile around the perimeter. `1.0` wraps one rainbow once around the whole edge, so adjacent bulbs differ only subtly. Raise it to `2` or `3` to tile multiple rainbows and widen the per-bulb color contrast.

<TomlExample
  code={`# Two full spectra around the ring ‚Äî punchier per-bulb color
border = { style = "lightbulbs", lit_color = "rainbow", hue_wraps = 2 }`}
/>

### When to use each style

`lightbulbs` reads as a **physical object** (a marquee of light fixtures). `rainbow` / `color_cycle` read as a **graphical effect** (a halo or outline). Pick the former for theatrical / vintage / sports-bar / Vegas-pastiche aesthetics; pick the latter for clean / modern / decorative work.

`lightbulbs` is unconditionally animated (`frame_invariant = False`) ‚Äî image-widget fast paths correctly force per-tick redraws on it, same as the other animated borders.

#### Common patterns

**Vegas / theatrical marquee** ‚Äî a fast, tightly-packed traveling lit window:

<TomlExample
  code={`border = { style = "lightbulbs", mode = "chase", speed_frames = 1, chase_density = 2 }
# Every other bulb lit, advancing one step per engine tick`}
/>

**Rainbow party lights** ‚Äî a calm static strand of colored bulbs, all lit at once:

<TomlExample
  code={`border = { style = "lightbulbs", mode = "chase", lit_color = "rainbow", chase_density = 1 }
# chase_density = 1 lights every bulb ‚Äî a steady rainbow strand`}
/>

**Holiday / themed two-color** ‚Äî pick a palette for lit and unlit, then alternate:

<TomlExample
  code={`border = { style = "lightbulbs", mode = "alternate", lit_color = [200, 30, 30], unlit_color = [20, 120, 40] }
# Red / green holiday twinkle`}
/>
