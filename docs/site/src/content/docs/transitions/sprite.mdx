The flair pack (`led-ticker-flair`) ships two categories of transition: **homage sprite-trails** â€” pixel-art characters that cross the panel â€” and **text-effect transitions** that manipulate the outgoing content itself. Both are referenced by their namespaced slug, e.g. `transition = "pokeball.forward"` or `transition = "flair.spinout"`.

## Homage transitions (flair pack)

The four homage families â€” **nyancat**, **pokeball**, **pacman**, and **sailor_moon** â€” ship together in the `led-ticker-flair` package (one install, four entry points, each with `.forward`, `.reverse`, and `.alternating` variants).

### Install

Add one line to `config/requirements-plugins.txt` (copied from `config/requirements-plugins.example.txt`):

```text
led-ticker-flair
```

One install registers all four families. See [Plugins](/plugins/) for the install flow.

A bare `transition = "nyancat"` (the pre-plugin spelling) raises a config-load error pointing at the namespaced `nyancat.forward` slug â€” the namespaced form is what the flair pack registers.

:::note[Existing configs need no change]
The type strings (`nyancat.forward`, `pokeball.alternating`, etc.) are unchanged â€” only the install source consolidates. Configs already using these transitions keep working as-is. If you previously pinned individual family tags (e.g. `@nyancat-v0.1.0`), those installs continue to work; switch to the single flair line at your convenience.
:::

## Hires

`nyancat.forward` and `pokeball.forward` auto-activate hi-res sprites whenever the canvas is at `default_scale > 1` â€” that is, any big / scaled sign (see [Display](/concepts/display/)), typically a bigsign-style build. The hi-res variants are noticeably more detailed and read better at distance:

- **nyancat** â€” animated webp Nyan Cat sprite at native physical resolution
- **pokeball** â€” rolling pokeball with animated run-cycle sprite

The `sailor_moon` and `pacman` families keep their 8-bit look at all scales â€” the chunky aesthetic is part of the design.

## Other plugin sprites

The rolling-**baseball** family (`baseball.roll`, `baseball.roll_reverse`, `baseball.roll_alternating`) ships with the **[baseball](https://github.com/JamesAwesome/led-ticker-plugins/tree/main/plugins/baseball)** package in the led-ticker-plugins monorepo, alongside that package's MLB widgets and `:baseball.ball:` emoji. Install it via `config/requirements-plugins.txt` (see [Plugins](/plugins/)) to use it.

## Text-effect transitions (flair pack)

### `flair.spinout` (flair >= 0.3.0, core >= 4.6)

`flair.spinout` spins the outgoing widget's content out of view â€” starting at rest, then accelerating â€” before cutting to the incoming widget. Unlike a sprite trail, nothing crosses the panel; the content itself rotates and disappears. Transition duration follows `[transitions] duration` (default 0.5 s; sprite-style durations of 1.0â€“1.5 s look best for a full revolution or two).

<DemoGif
  src="/demos-pinned/transition-flair-spinout.gif"
  caption="`flair.spinout` â€” outgoing text accelerates and spins out before the incoming widget appears"
/>

#### Install

`flair.spinout` is part of the same `led-ticker-flair` package as the homage sprite-trails:

```text
led-ticker-flair
```

One install registers all flair transitions and animations. See [Plugins](/plugins/) for the install flow.

#### Minimal config

<TomlExample
  title="Minimal"
  code={`[transitions]
default = "flair.spinout"
duration = 1.0

[[playlist.section]]
mode = "slideshow"
hold_time = 3.0

[[playlist.section.widget]]
type = "message"
text = "SPIN OUT"
center = true
font_color = [0, 220, 255]

[[playlist.section]]
mode = "slideshow"
hold_time = 2.5

[[playlist.section.widget]]
type = "message"
text = "AND CUT"
center = true
font_color = [80, 255, 80]`}
/>

#### Knobs

| Knob          | Default | Notes                                                   |
| ------------- | ------- | ------------------------------------------------------- |
| `revolutions` | `2`     | Integer >= 1. Full turns before the content disappears. |
| `direction`   | `"cw"`  | `"cw"` (clockwise) or `"ccw"` (counter-clockwise).      |

Set knobs via an inline table on `[transitions] default` or on the section's `transition` field:

```toml
[transitions]
default = {type = "flair.spinout", revolutions = 1, direction = "ccw"}
duration = 1.0
```

#### Behavior notes

- **Background color holds.** The outgoing section's `bg_color` fills the panel for the entire spin duration â€” the background never flashes or drops to black mid-transition.
- **Scrolled text spins from its parked position.** If the outgoing widget had been scrolling, `flair.spinout` picks up at the position where the text stopped (the same `outgoing_scroll_pos` continuity used by the push family).
- **Colors freeze during the spin.** Animated `font_color` providers (`rainbow`, `color_cycle`, `shimmer`) are paused for the outgoing widget while it spins, consistent with how transitions pause frame-aware effects in general.
- **Pairs with `flair.propeller`.** Use `flair.spinout` as the inter-widget or between-sections transition alongside `animation = "flair.propeller"` on `message` widgets for a full spin-in / spin-out cycle. See [flair.propeller â†’ Pairing with flair.spinout](/concepts/animations/#pairing-with-flairspinout).

### `flair.fireworks` (flair >= 0.7.0, core >= 4.10)

`flair.fireworks` holds the outgoing widget while staggered firework bursts launch as rising streaks from the bottom edge, then open into black "burn-through" holes ringed by radial spoke trails â€” an asterisk of bright white heads and fading colored tails, not a ring outline. Past the halfway point every burst's hole blooms in lockstep toward full-panel coverage, revealing the incoming widget through the expanding circles and blacking out everything outside their union, until embers fade and the whole panel cuts to the incoming section. Transition duration follows `[transitions] duration` â€” about 1.2 s gives the bursts room to stagger, open, and bloom without feeling rushed.

`flair.fireworks` is part of the same `led-ticker-flair` package as the homage sprite-trails and `flair.spinout`:

```text
led-ticker-flair
```

One install registers all flair transitions and animations. See [Plugins](/plugins/) for the install flow.

#### Minimal config

<TomlExample
  title="Minimal"
  code={`[transitions]

/>

With knobs, set via an inline table on the section's `transition` field (or on `[transitions] default`/`between_sections`):

<TomlExample
  title="With knobs"
  code={`[[playlist.section]]

/>

#### Knobs

| Knob     | Default                  | Notes                                                                                                                                                                                                                                                                                                                                  |
| -------- | ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `bursts` | size-adaptive (3-6)      | Number of firework bursts. Auto-picked from the panel's width-to-height ratio when omitted â€” wider panels get more bursts (smallsign: 5, bigsign: 4, longboi: 6) â€” or set explicitly in `[2, 8]`.                                                                                                                                      |
| `colors` | 8-color built-in palette | List of `[r, g, b]` triples, cycled across bursts in order. **Note:** this is `colors`, not `transition_colors` â€” core's `transition_colors` config field does not reach plugin transitions (a known gap); `colors` is this transition's own constructor kwarg.                                                                        |
| `seed`   | `None` (OS entropy)      | Fixes the RNG for reproducible bursts â€” useful for regression tests and reproducible demo GIFs. Not a promise that the same seed plans the same sequence forever: a re-fire of the same running instance continues drawing from that same RNG stream, so consecutive firings on a live sign still vary. Leave unset for a real deploy. |

#### Behavior notes

- **Size-adaptive by default.** Burst count, radius, and stagger timing are all derived from the panel's actual width/height at draw time â€” the same `transition = "flair.fireworks"` line looks right on smallsign, bigsign, and longboi without per-sign tuning.
- **Two-phase design, painted through black.** There is no `GetPixel` on real hardware (the framebuffer stores pre-computed GPIO bitplane data, not readable RGB), so nothing can be composited from a previous frame â€” every frame is drawn from scratch from the widgets plus burst geometry. Phase one (`t < 0.5`) plays over the outgoing widget; phase two (`t >= 0.5`) switches to the incoming widget and blacks out everything outside the union of the (still-growing) burst circles, guaranteeing full coverage by `t == 1.0` regardless of any individual burst's stagger.
