Sprite transitions run a pixel-art character across the panel that erases the outgoing widget and reveals the incoming one — themed and fun. They are provided by the external **[led-ticker-arcade](https://github.com/JamesAwesome/led-ticker-arcade)** plugin: four families (`arcade.nyancat`, `arcade.pokeball`, `arcade.pacman`, `arcade.sailor_moon`), each with forward, reverse, and alternating variants. Reference them by their namespaced slug, e.g. `transition = "arcade.pokeball"`.

## Install

Add the plugin to `config/requirements-plugins.txt` (copied from `config/requirements-plugins.example.txt`):

```
git+https://github.com/JamesAwesome/led-ticker-arcade.git@main
```

It registers its transitions and the `:arcade.pokeball:` emoji at startup. See [Plugins](/plugins/) for the install flow. A bare `transition = "nyancat"` (the pre-plugin spelling) now raises a config-load error pointing at the namespaced `arcade.nyancat` slug.

## Hires

`arcade.nyancat` and `arcade.pokeball` auto-activate hi-res sprites whenever the canvas is at `default_scale > 1` — that is, any big / scaled sign (see [Display](/concepts/display/)), typically a bigsign-style build. The hi-res variants are noticeably more detailed and read better at distance:

- **arcade.nyancat** — animated webp Nyan Cat sprite at native physical resolution
- **arcade.pokeball** — procedural ball with animated Pikachu run-cycle sprite

`arcade.sailor_moon` and `arcade.pacman` keep their 8-bit look at all scales — the chunky aesthetic is part of the design.

## Other plugin sprites

The rolling-**baseball** family (`baseball.roll`, `baseball.roll_reverse`, `baseball.roll_alternating`) ships with the external **[led-ticker-baseball](https://github.com/JamesAwesome/led-ticker-baseball)** plugin, alongside that plugin's MLB widgets and `:baseball.ball:` emoji. Install it via `config/requirements-plugins.txt` (see [Plugins](/plugins/)) to use it.
