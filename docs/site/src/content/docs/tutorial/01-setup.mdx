Over the next five chapters you'll build a real bigsign config from scratch —
starting with a single message and ending with a multi-section sign that rotates
through a rainbow welcome card, a brand card (the @firebirdyoga.demo handle next
to the hi-res Instagram sprite over a phoenix image with scrolling promo copy),
and a held info section (Summer Camps, Open daily, All levels).

This chapter covers getting the code running on your laptop. No hardware needed.

:::note[Following along on a smaller sign?]
This tutorial uses the bigsign as its worked example, but the config steps —
the `[display]` block, sections, widgets, transitions — apply just as well to
the cheaper [smallsign](/hardware/smallsign/), the obvious first build. You
don't need the 8-panel rig to follow along; only the panel knobs differ.
:::

**~10 min · no hardware needed**

**What you'll need:**

- A laptop (macOS, Linux, or Windows)
- [Docker](https://docs.docker.com/get-docker/) — for `make try` (optional but the fastest path to a live preview)
- Or: the repository cloned and `make dev` run for the GIF-renderer path (both covered below)
- No LED panel or Raspberry Pi — everything previews in Docker or via `make render-demo`

## Prerequisites

:::note[On a Raspberry Pi or an older system?]
Use the Docker path (`make try` / `make setup`) — it handles Python for you.
The `make dev` path here needs Python 3.14 on your machine
([python.org/downloads](https://www.python.org/downloads/); install uv with
`pip install uv`).
:::

You'll need the following installed before you start:

- **Git** — to clone the repository
- **Python 3.14+** — led-ticker targets the latest stable CPython
- **[uv](https://docs.astral.sh/uv/)** — the package manager used throughout the project (`pip install uv` or `brew install uv`)
- **make** — standard on macOS and Linux; on Windows use WSL or Git Bash

## Clone and install

```bash
git clone https://github.com/JamesAwesome/led-ticker.git
cd led-ticker
make dev
```

What each command does:

- **`git clone`** — downloads the repository, including all bundled fonts, example
  configs, and the laptop renderer.
- **`cd led-ticker`** — enters the project root. All `make` targets and file paths
  in this tutorial assume you're running from here.
- **`make dev`** — runs `uv sync --extra dev` to install the Python package and all
  dev dependencies into a local virtual environment, then installs the pre-commit
  hooks. This does **not** require the `rgbmatrix` C library — that's only needed on
  a real Raspberry Pi. Laptop development works entirely through a built-in stub.

## Render the destination config

Before writing a single line of your own TOML, render the tutorial destination
config so you can see what "done" looks like:

```bash
make render-demo CONFIG=config/config.bigsign.firebird.example.toml OUT=preview.gif
open preview.gif          # macOS
xdg-open preview.gif      # Linux
```

`make render-demo` runs the same display engine that drives the real panels, but
writes each frame to a GIF instead of pushing it to hardware. The output is at
native panel resolution (256×64 logical pixels, scaled up 4× by the renderer for
readability).

You should see a looping GIF that cycles through three sections — a rainbow
welcome card, a brand card with the Instagram sprite + phoenix image + scrolling
promo, and a held info section (Summer Camps, Open daily, All levels):

Open `config/config.bigsign.firebird.example.toml` in your editor and skim it.
You'll recognize each section in the GIF. The comments explain the Pi-specific
knobs (`pixel_mapper_config`, `gpio_slowdown`, `pwm_bits`) — those are safe
to ignore for now; the laptop renderer skips them silently.

## What this chapter doesn't cover

This tutorial stays on the laptop renderer throughout. Deploying to a real Pi —
wiring up the HAT, running `make setup`, and the panel bring-up workflow — is
covered separately in [Building your own](/hardware/building-your-own/). Come
back to that guide once you've finished the five chapters and have a config
you're happy with.
