The `calendar` widget pulls upcoming events from any subscribed iCal (`.ics`) URL and displays them on the panel. It works like the `rss_feed` widget — a background task polls the feed and builds one scrolling message per event; the display loop picks up changes on the next pass without restarting.

It reuses the same text-render surface as `message` and `clock`, so [`font_color`](/concepts/color-providers/), [`border`](/concepts/borders/), and font knobs all work the same way. Each line is drawn in two colors by default — the time phrase in `time_color` (amber) and the event title in `font_color` (white); see [Colors](#colors).

## Layouts

### `agenda` (default)

Each upcoming event becomes its own scrolling line. Events are sorted chronologically; the day label updates automatically (`Today`, `Tomorrow`, weekday name, or `Mon D` for dates further out). The time phrase and the title are drawn in separate colors — see [Colors](#colors).

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
layout = "agenda"
max_events = 5
lookahead_days = 7`}
/>

### `next`

A single live countdown line to the soonest upcoming event: `"Standup in 25m"` / `"Dentist in 2h 10m"` / `"Trip in 3d"` / `"Standup now"`. The engine redraws every 50 ms, so the countdown ticks in real time.

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
layout = "next"
empty_text = "All clear"`}
/>

### `two_row`

Each event becomes a two-row card: the **held top row** shows the day + time (`Tomorrow 3:00 PM`; all-day events show just the day), and the **bottom row** shows the title, scrolling only if it overflows. Cards rotate one-per-screen like `agenda`. This trades horizontal scrolling for vertical space — good when titles are long and you'd rather read them on their own line than watch them scroll.

The top row uses `time_color`, the bottom row uses `font_color` — the same colors as the other layouts (a highlighted event renders the whole card in `highlight_color`). Tune the split with `top_row_height`.

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
layout = "two_row"
time_color = [255, 200, 60]
font_color = [255, 255, 255]`}
/>

<Aside>
  Two rows split the canvas, so each row's band is at most 8 logical rows (`content_height` caps at
  16 on both reference signs). The standard `6x12` font is too tall for that, so `two_row`
  automatically uses the band-sized `5x8` when you don't set `font`. For larger or hi-res text,
  choose a `font` / `font_size` whose line-height fits the row — [`led-ticker
  validate`](/reference/cli/) rejects one that's too tall (naming the row) before you deploy. Raise
  `content_height` or set `top_row_height` for an asymmetric split if you need more room.
</Aside>

## Filter & highlight

`filter` keeps only events whose summary contains any keyword (case-insensitive). An empty list shows all events.

`highlight` recolors matching events with `highlight_color` (amber by default) and guarantees they appear even if they would otherwise be dropped by the `max_events` cap.

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
filter = ["standup", "1:1", "review"]
highlight = ["1:1"]
highlight_color = [255, 200, 60]
max_events = 4`}
/>

## Colors

Each line is drawn in **two colors**: the time/relative phrase in `time_color` (amber `[255, 200, 60]` by default) and the event title in `font_color` (white by default). This is the default look — the "when" pops while the title stays legible.

- In **agenda** mode the time phrase is the leading `Tomorrow 3:00 PM ·`; the title is everything after.
- In **next** mode the title comes first and the trailing `· in 5m` is the time phrase.

Both accept the full [color-provider](/concepts/color-providers/) range — a constant `[r,g,b]`, `"rainbow"`, `"color_cycle"`, or a `{style="gradient", …}` table — so you can animate either part independently.

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
time_color = [255, 200, 60]   # amber time phrase (default)
font_color = [255, 255, 255]  # white title (default)`}
/>

To use a **single color** for the whole line, set both to the same value:

<TomlExample
  code={`time_color = "rainbow"
font_color = "rainbow"`}
/>

A line whose summary matches [`highlight`](#filter--highlight) ignores both and renders entirely in `highlight_color` — a highlight is a whole-line attention state, so two-tone applies to non-highlighted lines only.

## Timezone & format

`timezone` overrides which timezone events are displayed in. Set it to any [IANA timezone name](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones). Without it, the system local timezone is used.

`time_format` controls how the event time appears in agenda lines and the held top (when) row of `two_row` cards:

- `"12h"` (default) — `3:00 PM` (no leading zero)
- `"24h"` — `15:00` (zero-padded)

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
timezone = "America/New_York"
time_format = "12h"`}
/>

## Getting your .ics URL

### Google Calendar

1. Open [calendar.google.com](https://calendar.google.com) → Settings → choose your calendar → **Integrate calendar**.
2. Copy the **"Secret address in iCal format"** link (ends in `.ics`).

<Aside type="caution">
  The Google Calendar secret address is a credential — anyone with the URL can read your calendar.
  Your `config.toml` is gitignored, so put the URL directly there. Do not commit it to a public
  repository or paste it where others can see it. (led-ticker has no environment-variable
  interpolation in config files, so `.env` references won't work.)
</Aside>

### iCloud

In the iCloud Calendar web app, right-click a calendar → **Share Calendar** → enable **Public Calendar** → copy the link. Replace `webcal://` with `https://` so led-ticker can fetch it.

### Other calendars

Any service that provides a public or authenticated `.ics` subscription URL works — Fastmail, ProtonMail Calendar, Nextcloud Calendar, and most corporate calendar systems all support this standard format.

## `file://` and local feeds

You can point `ics_url` at a local file on the Pi using a `file://` URL:

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "file:///home/pi/my-calendar.ics"
timezone = "UTC"`}
/>

<Aside>
  Prefer an absolute path (starting with `/`) for deployed configs. A relative path like
  `file://cal.ics` resolves against the working directory of the process, which can vary depending
  on how led-ticker is launched (systemd, Docker, or manual).
</Aside>

This is useful for syncing an `.ics` export from a local calendar app, or for testing with a committed sample feed. You can keep the file current with a cron job that fetches the feed and writes it to disk — led-ticker polls it on its normal `update_interval`.

## Troubleshooting

If the feed can't be loaded, led-ticker fails gracefully instead of crashing:

- **Catch it before it runs.** [`led-ticker validate`](/reference/cli/) flags an unfilled placeholder `ics_url` as an error, and **warns** if a `file://` / local-path feed doesn't exist yet (a warning, not an error — a cron job may write the file later).
- **On the panel.** A first-load failure shows `error_text` (default `"Calendar unavailable"`) rather than `empty_text`, so a broken feed is visibly different from a calendar that simply has no upcoming events. A later transient failure keeps the last events that loaded successfully.
- **In the logs.** The failure is logged as a single, plain-language line that names the cause and points back here — e.g. `Calendar feed file not found: '…'`, `Calendar feed returned HTTP 404 for …`, or `Calendar feed unreachable: …`. (The full traceback is logged at `DEBUG` only.)

<Aside type="tip">
  The most common cause is an `ics_url` that was never filled in, or an iCloud link still using the
  `webcal://` scheme — led-ticker rewrites `webcal://` to `https://` automatically, but a copy-paste
  typo in the URL will surface as an HTTP error in the log.
</Aside>

### Highlighted events always visible

Events matching `highlight` always appear in the agenda even when there are more than `max_events` events in the window — they are never dropped by the cap.

<TomlExample
  code={`[[playlist.section.widget]]
type = "calendar"
ics_url = "https://example.com/cal.ics"
highlight = ["payday", "release"]
highlight_color = [255, 200, 60]
max_events = 3`}
/>
