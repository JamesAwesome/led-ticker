The `coinbase` widget fetches the current spot price for a cryptocurrency pair from [Coinbase's public API](https://api.coinbase.com) and displays the price alongside the 24-hour change percentage. No API key is required. The change field is automatically colored green when the price is up vs the previous fetch, red when down, and gray when unchanged.

## Common patterns

A single coin display — common symbols include `BTC-USD`, `ETH-USD`, `SOL-USD`, and `DOGE-USD`:

  code={`[[playlist.section.widget]]

/>

Multiple coins in one section — each widget shows its own pair, and they scroll one after the other:

<TomlExample code={`[[playlist.section.widget]]

Slower poll for a low-traffic display — the default 5-minute interval is already conservative, but you can extend it further:

<TomlExample
  code={`[[playlist.section.widget]]
type = "coinbase"
symbol = "BTC-USD"
currency = "USD"
update_interval = 600`}
/>

## Pitfalls

**Trend color compares against the previous fetch, not the day's open.** The green/red coloring is driven by the difference between the most recent price and the price from the fetch before it. Two back-to-back fetches during a calm market can flip green/red on tiny sub-cent drift. This is expected behavior, not a bug.

**Coinbase's public API can throttle aggressive polling.** The default `update_interval` of `300` seconds (5 minutes) is intentionally conservative. Lowering it substantially on a high-traffic display is not recommended and may cause the widget to return stale data silently.
