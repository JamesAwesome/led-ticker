The `coingecko` widget is essentially `coinbase` pointed at the [CoinGecko API](https://api.coingecko.com) — same on-screen layout (symbol · price · change%), same trend coloring, different data source. CoinGecko covers a much broader universe of coins than Coinbase, making it the right choice for altcoins that aren't listed on Coinbase. No API key is required for the public tier.

The key difference from `coinbase`: CoinGecko identifies coins by a slug ID (`"bitcoin"`) rather than a trading pair (`"BTC-USD"`), so the config requires both a display `symbol` and a lookup `symbol_id`.

## Common patterns

## Tips

### `symbol_id` must be the CoinGecko coin ID, not the ticker

Searching for `"btc"` will fail — the correct ID is `"bitcoin"`. Check the full list at [coingecko.com/en/coins/list](https://www.coingecko.com/en/coins/list). Common IDs: `bitcoin`, `ethereum`, `solana`, `dogecoin`, `pepe`, `dogwifcoin`.

### CoinGecko's free tier rate-limits at ~10–30 calls per minute

If you run many `coingecko` widgets with a low `update_interval`, you may hit the limit and receive error responses. The default 5-minute interval is safe for up to ~6 simultaneous widgets; for more, raise `update_interval` proportionally.

### Trend color compares against the previous fetch, not the day's open

The green/red coloring reflects the price difference between the two most recent polls. Near-identical prices between fetches can flip color on tiny drift. This is expected behavior.
