The `gif` widget plays animated images at native panel resolution — each LED maps to one pixel in the source, not a scaled block. Format-agnostic: anything Pillow opens with `n_frames` and `seek()` works (`.gif`, animated `.webp`, `.apng`, multi-frame `.tiff`). Per-frame durations come from the file's own metadata.

An optional text overlay can be added without any additional configuration: set `text` and it appears on the panel alongside the animation. When the text is longer than the canvas, set `text_align = "scroll"` or `"scroll_over"` for a marquee. For two-row layouts (held top + scrolling bottom), set `bottom_text` to switch modes.

## Common patterns

Silent gif (no text):

<TomlExample code={`[[playlist.section.widget]]
type = "gif"
path = "config/assets/my_animation.gif"
fit = "pillarbox"
image_align = "center"
gif_loops = 3`} />

Scrolling marquee over the image (`scroll_over` — text in front):

<TomlExample code={`[[playlist.section.widget]]
type = "gif"
path = "config/assets/pika_wave_transparent.gif"
fit = "pillarbox"
image_align = "left"
text = "Welcome to the show!"
text_align = "scroll_over"
font_color = [255, 200, 100]
gif_loops = 999`} />

Scrolling text behind a transparent silhouette (`scroll` — text walks under the image):

<TomlExample code={`[[playlist.section.widget]]
type = "gif"
path = "config/assets/pika_wave_transparent.gif"
fit = "pillarbox"
image_align = "left"
text = "Scrolling behind Pika!"
text_align = "scroll"
font_color = [200, 200, 255]
gif_loops = 5`} />

Two-row text overlay (set `bottom_text` to switch modes — image plays underneath):

<TomlExample code={`[[playlist.section.widget]]
type = "gif"
path = "config/assets/pika_wave_transparent.gif"
fit = "pillarbox"
image_align = "right"
top_text = "@moonbunny"
top_color = [225, 48, 108]
bottom_text = "Aerial circus · all levels welcome"
bottom_color = [255, 240, 200]
gif_loops = 3`} />

## Pitfalls

<DecisionRule id="14" />

**`text_align="scroll"` + `fit="stretch"` is rejected at config-load.** The `scroll` mode exposes scrolling text through the transparent / pillarbox regions of the image. When `fit="stretch"`, the image fills the entire panel with no transparent or black regions — there is nothing for the text to show through. Use `text_align="scroll_over"` for a marquee over a fullscreen image.

**`text_x_offset` is rejected with scroll modes.** The offset only applies to static text placement (`text_align = "left"` or `"right"`). With `scroll` or `scroll_over`, the marquee position is driven by the scroll logic; a constant x offset would skew the entry point in a confusing way.
