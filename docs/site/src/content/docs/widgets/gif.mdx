The `gif` widget plays animated images at native panel resolution — each LED maps to one pixel in the source, not a scaled block. Format-agnostic: anything Pillow opens with `n_frames` and `seek()` works (`.gif`, animated `.webp`, `.apng`, multi-frame `.tiff`). Per-frame durations come from the file's own metadata.

An optional text overlay can be added without any additional configuration: set `text` and it appears on the panel alongside the animation. When the text is longer than the canvas, set `text_align = "scroll"` or `"scroll_over"` for a marquee. For two-row layouts (held top + scrolling bottom), set `bottom_text` to switch modes.

## Pitfalls

### Typewriter is single-row only

`animation = "typewriter"` needs fixed-position glyphs, so it can't combine with `bottom_text` (two-row layout has no fixed positions), with `text_align = "scroll"` / `"scroll_over"` (the cursor moves each tick), or with an empty `text` field. Drop the conflicting setting and the typewriter still composes cleanly with `font_color = "rainbow"` and `border = "rainbow"` — independent counters, all animate together.

### `text_align="scroll"` requires a non-opaque image

The `scroll` mode displays scrolling text through transparent / pillarbox regions of the image. With `fit="stretch"` the image fills every pixel and there are no transparent regions for the text to walk behind — so the text is always hidden. Use `text_align="scroll_over"` for a marquee over a fullscreen image, or use a `.gif` with transparency and `fit="pillarbox"`.

### `text_x_offset` is rejected with scroll modes

`text_x_offset` only applies to static text placement (`text_align = "left"` or `"right"`). With `scroll` or `scroll_over`, the marquee's x position is driven by the scroll logic; adding a constant offset would skew the entry point in a confusing way. Remove the offset, or switch to a static `text_align`.
