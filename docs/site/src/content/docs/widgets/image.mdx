The `image` widget displays a single still image at native physical resolution — each LED maps to one pixel in the source. Use it for logos, product photos, brand artwork, or any single-frame graphic. This widget is essentially `gif` with a single frame: it shares the `gif` widget's full feature surface (fit modes, text overlay, two-row, animation, border) minus the per-frame mechanics — the only widget-specific knob is `hold_seconds` instead of `gif_loops`.

## Common patterns

Static logo (no text):

<TomlExample
  code={`[[playlist.section.widget]]
type = "image"
path = "config/assets/my_logo.png"
fit = "pillarbox"
image_align = "center"
hold_seconds = 6.0`}
/>

Image with scrolling text overlay (`scroll_over` — text in front of the image):

<TomlExample
  code={`[[playlist.section.widget]]
type = "image"
path = "config/assets/bunny-transparent.png"
fit = "pillarbox"
image_align = "left"
text = "Follow us on Instagram!"
text_align = "scroll_over"
hold_seconds = 5.0
font_color = [200, 100, 255]`}
/>

Scrolling text behind a transparent silhouette (`scroll` — text walks under the image; requires a PNG with transparency):

<TomlExample
  code={`[[playlist.section.widget]]
type = "image"
path = "config/assets/bunny-transparent.png"
fit = "pillarbox"
image_align = "left"
text = "Scrolling behind the bunny!"
text_align = "scroll"
font_color = [200, 200, 255]`}
/>

Two-row text overlay (set `bottom_text` to switch modes — image sits underneath):

<TomlExample
  code={`[[playlist.section.widget]]
type = "image"
path = "config/assets/my_logo.png"
fit = "pillarbox"
image_align = "right"
top_text = "@myhandle"
top_color = [225, 48, 108]
bottom_text = "Open daily 10am–8pm"
bottom_color = [255, 240, 200]
hold_seconds = 6.0`}
/>

## Pitfalls

<DecisionRule id="14" />

**`text_align="scroll"` + `fit="stretch"` is rejected at config-load.** The `scroll` mode exposes scrolling text through the transparent / pillarbox regions of the image. When `fit="stretch"`, the image fills the entire panel with no transparent or black regions — there is nothing for the text to show through. Use `text_align="scroll_over"` for a marquee over a fullscreen image, or use a PNG with transparency and `fit="pillarbox"`.

**`text_x_offset` is rejected with scroll modes.** The offset only applies to static text placement (`text_align = "left"` or `"right"`). With `scroll` or `scroll_over`, the marquee position is driven by the scroll logic; a constant x offset would skew the entry point in a confusing way.

**`hold_seconds` must be at least `0.05`.** Values below this minimum are rejected at config-load.

**Animated source files only show frame 0.** If you pass an animated `.gif` or `.webp` to the `image` widget, only the first frame is decoded and displayed. Use the `gif` widget for animation.
