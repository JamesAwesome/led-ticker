The `mlb_standings` widget fetches overall MLB league standings from the free StatsAPI and displays them as a scrolling list. It shows the top-N teams by overall rank and always includes a configurable set of tracked teams — so your team stays visible even when they're buried in the middle of the pack. During the offseason (before any games have been played), the widget automatically switches to a pre-season message with the opening-day date, then resumes showing standings once the season begins. No API key is required.

## Common patterns

Top-3 only — no tracked teams, just the league leaders:

<TomlExample
  code={`[[playlist.section.widget]]
type = "mlb_standings"
teams = []
top_n = 3
title = "MLB Top 3"`}
/>

Tracked teams only — skip the overall leaders and show only the teams you care about:

<TomlExample
  code={`[[playlist.section.widget]]
type = "mlb_standings"
teams = ["NYM", "NYY"]
top_n = 0`}
/>

Top-5 leaders plus two tracked teams, with a dark background and hourly updates:

<TomlExample
  code={`[[playlist.section.widget]]
type = "mlb_standings"
teams = ["BOS", "LAD"]
top_n = 5
bg_color = [0, 40, 80]
update_interval = 3600`}
/>

## Tips

### Tracked teams dedup with top-N

If a tracked team already appears in the top-N list, the widget shows them there and skips the duplicate. You won't see the same team twice.

### Offseason and pre-season behavior

When the current season's standings all show 0-0 (no games played yet), the widget fetches the upcoming schedule and shows an opening-day message like `Opens Mar 27`. Between the World Series and Spring Training, the MLB StatsAPI still returns the prior season's final standings — so the widget keeps displaying those rather than going blank. Expect last-season data from roughly mid-October through late March.

### Poll interval defaults to daily

Standings shift slowly — the default `update_interval` of 86400 s (24 hours) is appropriate for most displays. During a tight pennant race in September, `update_interval = 3600` gives hourly refreshes without being rude to the public API. Dropping below 60 s is not recommended.
