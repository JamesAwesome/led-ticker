The **weather.current** widget is provided by the **[weather](https://github.com/JamesAwesome/led-ticker-plugins/tree/main/plugins/weather)** package in the led-ticker-plugins monorepo, referenced in config as `type = "weather.current"`. It shows current conditions and temperature from [WeatherAPI.com](https://www.weatherapi.com/) with a pixel-art condition icon. On hires (scaled) displays the condition icons render richer Noto-derived sprites since core 4.22.0; small (scale-1) signs keep the original hand-drawn 8×8 icons.

Install the plugin and set your API key, then add a widget block to your config:

```toml
[[playlist.section]]
mode = "slideshow"

[[playlist.section.widget]]
type = "weather.current"
text = "NYC"
location = "New York"
units = "imperial"
```

To install the plugin on a Docker deploy, add its line to the plugin manifest and restart:

```bash
# First plugin? Create your manifest from the example:
cp -n config/requirements-plugins.example.txt config/requirements-plugins.txt
# Add the package (one name per line), then restart:
echo "led-ticker-weather" >> config/requirements-plugins.txt
docker compose restart
```

You also need `WEATHERAPI_KEY` in your `.env` — free key at [weatherapi.com](https://www.weatherapi.com/). Other install paths (web UI Store tab, dev checkout) are on the [Plugins page](/plugins/).

**Full documentation** — all options, config examples, and a demo gif — lives in the [weather package README](https://github.com/JamesAwesome/led-ticker-plugins/blob/main/plugins/weather/README.md).
