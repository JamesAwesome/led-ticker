The `weather` widget fetches current conditions and temperature from [WeatherAPI.com](https://www.weatherapi.com/) and displays them with a pixel-art icon for the condition — sun, cloud, rain, snow, thunder, fog, or partly cloudy. It polls in the background via async retry so the display keeps running even if the API is temporarily unreachable.

:::note[No demo gif]
The weather widget makes a live HTTP call to WeatherAPI.com on every visit, which requires a `WEATHERAPI_KEY` not available in CI. Run a `weather` widget locally against your own API key to see it animate.
:::

<TomlExample title="Minimal example" code={`[[playlist.section.widget]]

You'll need a free API key from [WeatherAPI.com](https://www.weatherapi.com/) — set `WEATHERAPI_KEY` in your `.env` file.

## Common patterns

Two-color layout — gray label, bright temperature:

<TomlExample code={`[[playlist.section.widget]]
type = "weather"
message = "NYC"
location = "New York"
units = "imperial"
font_color = [150, 150, 150]
font_color_temp = [255, 255, 255]`} />

Text-only display — no pixel icon, shows the condition string instead:

<TomlExample code={`[[playlist.section.widget]]
type = "weather"
message = "Weather"
location = "10001"
units = "imperial"
show_icon = false`} />

Slower poll for stable outdoor forecasts — fetch every 30 minutes:

<TomlExample code={`[[playlist.section.widget]]
type = "weather"
message = "Outside"
location = {lat = 40.71, lon = -74.01}
units = "metric"
update_interval = 1800`} />

## Pitfalls

**`WEATHERAPI_KEY` must be set before startup.** The widget raises `ValueError` at the first poll if the env var is missing. Add it to your `.env` file (see `.env.example`) and restart the container.

**Default poll interval is 3 hours.** The `update_interval` default of `10800` seconds is intentionally conservative — conditions don't change that fast and the WeatherAPI free tier has monthly request quotas. Setting below 60 seconds on a busy sign will exhaust the quota. For a "is it raining right now?" display, `update_interval = 300` (5 minutes) is a reasonable compromise.

**Location format matters.** If a city name is ambiguous (e.g. `"Springfield"`), WeatherAPI.com may return the wrong city. Prefer the `{lat = ..., lon = ...}` table form or a ZIP code for precision.
