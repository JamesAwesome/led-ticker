# pnpm Migration + PR-Only Cloudflare Builds Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Switch `docs/site/` from npm to pnpm using corepack, and gate Cloudflare Pages so only PR-associated branches produce preview deploys (random feature-branch pushes don't waste builds).

**Architecture:** pnpm declared via the `packageManager` field in `docs/site/package.json` so corepack activates it automatically in any environment that supports corepack (Cloudflare Pages, GitHub Actions, local dev). Single lock file: `pnpm-lock.yaml` replaces `package-lock.json`. PR-only builds: turn off Cloudflare's automatic preview deploys (set "Preview deployments" to None in the dashboard), and replace it with a `.github/workflows/docs-pr-preview.yml` workflow that triggers on `pull_request` events and uses `cloudflare/wrangler-action` to deploy to Cloudflare Pages with the PR's branch name. Production deploys on `main` continue to use Cloudflare's GitHub integration directly (auto-deploy on push).

**Tech Stack:** Node LTS + pnpm (latest stable, pinned via `packageManager`); Astro (existing); GitHub Actions + cloudflare/wrangler-action@v3 for PR previews.

**Context:** This is a follow-up to PR #14 (`worktree-feat-docs-site` branch). The work in this plan adds new commits to the SAME branch — they'll appear on the existing PR automatically when pushed.

### Step 1: Update `docs/site/cloudflare-build.sh`

Read the current file:

```bash
cat docs/site/cloudflare-build.sh
```

Replace its contents with:

```bash
#!/usr/bin/env bash
# Cloudflare Pages build entry point.
#
# Cloudflare's build env provides Node and Python out of the box, but
# not `uv` or pre-activated pnpm. We:
#   1. Install uv via the official script
#   2. Enable corepack and activate the pnpm version pinned in
#      docs/site/package.json's `packageManager` field
#   3. Sync Python deps so the gif renderer can import led_ticker
#   4. pnpm install + pnpm run build (which prebuilds demo gifs via
#      the build-demos.mjs script that shells out to uv run python)
#
# Set this as the build command in the Cloudflare Pages dashboard:
#   bash docs/site/cloudflare-build.sh
#
# Build output directory: docs/site/dist
# Production branch: main

set -euo pipefail

echo "[cloudflare-build] installing uv"
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"

echo "[cloudflare-build] uv version: $(uv --version)"

echo "[cloudflare-build] enabling corepack"
corepack enable

echo "[cloudflare-build] syncing Python deps"
uv sync

echo "[cloudflare-build] installing Node deps for the docs site (pnpm)"
cd docs/site
pnpm install --frozen-lockfile

echo "[cloudflare-build] building docs site (renders demo gifs first)"
pnpm run build

echo "[cloudflare-build] done. Output in docs/site/dist"
```

The key changes from the prior version: drop `npm ci` for `pnpm install --frozen-lockfile`, drop `npm run build` for `pnpm run build`, add `corepack enable` step. `corepack enable` is enough — it'll pick up the pnpm version from `packageManager` in package.json automatically.

## Task 3: Add PR-only Cloudflare deploy workflow

**Files:**
- Create: `.github/workflows/docs-pr-preview.yml`

### Step 1: Create the GH Actions workflow

```bash
mkdir -p .github/workflows
```

Create `.github/workflows/docs-pr-preview.yml` with this content:

```yaml
name: docs-pr-preview

on:
  pull_request:
    types: [opened, synchronize, reopened]
    paths:
      - "docs/**"
      - "tools/render_demo/**"
      - ".github/workflows/docs-pr-preview.yml"

permissions:
  contents: read
  pull-requests: write
  deployments: write

concurrency:
  group: docs-pr-preview-${{ github.head_ref }}
  cancel-in-progress: true

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true

      - uses: actions/setup-python@v5
        with:
          python-version: "3.13"

      - name: Install Python deps
        run: uv sync

      - uses: actions/setup-node@v4
        with:
          node-version: "lts/*"

      - name: Enable corepack
        run: corepack enable

      - name: Install Node deps
        working-directory: docs/site
        run: pnpm install --frozen-lockfile

      - name: Build site (renders demo gifs first)
        working-directory: docs/site
        run: pnpm run build

      - name: Deploy preview to Cloudflare Pages
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: pages deploy docs/site/dist --project-name=led-ticker --branch=${{ github.head_ref }}
          gitHubToken: ${{ secrets.GITHUB_TOKEN }}
```

Notes:
- `paths:` filter scopes the workflow to actually-relevant changes; a PR that only edits Python files outside `tools/render_demo/` won't trigger a docs preview build (saves CI minutes).
- `concurrency` cancels in-flight preview builds when a PR gets a new push, so only the latest commit's preview is deployed.
- `gitHubToken: ${{ secrets.GITHUB_TOKEN }}` lets `wrangler-action` post a deployment status on the PR (visible in the PR's checks UI) and comment with the preview URL.
- `--branch=${{ github.head_ref }}` scopes the Cloudflare deploy under the PR's branch name so each PR gets a distinct preview URL pattern.
