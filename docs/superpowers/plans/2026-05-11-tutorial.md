# Tutorial Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a five-chapter, hand-holding tutorial to the docs site that walks a moderately-technical reader from `git clone` to a polished bigsign preview gif of a fictional moonbunny aerial circus storefront sign.

**Architecture:** Multi-page Starlight tutorial under `/tutorial/`, sidebar group between Getting Started and Widgets. Each chapter is a single MDX file with inline `<DemoGif>` previews rendered from source TOMLs in `docs/site/demos-long/`. A new `TutorialNav.astro` component handles Prev/Next chaining. Custom font (Atkinson Hyperlegible) is gitignored; rendered locally before commit.

**Tech Stack:** Astro Starlight v0.39.x, MDX, existing `DemoGif` / `TomlExample` / `RelatedPages` components, existing `make render-long-demo` Makefile target.

**Source spec:** `docs/superpowers/specs/2026-05-11-tutorial-design.md`

- [ ] **Step 1: Create the moonbunny reference config**

Write `config/config.bigsign.moonbunny.example.toml`. Use `Inter-Bold` (bundled — no font setup needed for the reader at this stage). The config should exercise the concepts that the tutorial will teach across all five chapters, so the reader recognizes the patterns when they meet them in chapters 2-5.

Content guidelines:
- `[display]` block: 256×64 bigsign at default_scale=4. Include `pixel_mapper_config` Remap string + `gpio_slowdown` so it's a real-deploy-shaped config; the chapter explains "these are Pi-only knobs, ignore for now."
- At least 3 sections demonstrating different modes:
  - A `swap` section: `[playlist.section.title]` = "MOONBUNNY", with a two_row widget for `@moonbunny :instagram:` + scrolling promo
  - A `forever_scroll` section: a few short messages mixing types (hours, special promo, weather placeholder)
  - A `swap` section with an image widget (moon-transparent.png) + transition
- Use `Inter-Bold` for hi-res text, `font_size = 22` or `28` for legibility

Aim for ~50-70 lines of TOML. The reader will see this rendered in Chapter 1's preview gif as their destination.

- [ ] **Step 2: Create the Chapter 1 demo source TOML**

Write `docs/site/demos-long/tutorial-01-setup.toml`. This should render the moonbunny reference config. Two options:
- (a) Symlink: `ln -s ../../../config/config.bigsign.moonbunny.example.toml docs/site/demos-long/tutorial-01-setup.toml` (avoids duplication)
- (b) Copy the content with a `# render-duration: <N>` comment at top

The Makefile `render-long-demo` target uses `grep -E '^# render-duration:'` to pick up the duration, so option (a) requires the symlink target to also have the comment. Option (b) is simpler — just duplicate the file with the duration comment. **Decision: option (b).**

The `# render-duration:` should be long enough to capture one full cycle of the moonbunny config. Estimate: 3 sections × (~4-6 sec each including transitions) ≈ 20-25 sec. Start with `# render-duration: 30` and adjust after first render.
